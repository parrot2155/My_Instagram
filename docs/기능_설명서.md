# My Instagram 기능 설명서

작성일: 2026년 1월 22일

---

## 📱 프로젝트 개요

Instagram과 유사한 소셜 미디어 웹 애플리케이션입니다. 사용자는 회원가입 후 로그인하여 사진과 함께 게시물을 작성하고, 다른 사용자들의 게시물을 피드에서 확인할 수 있습니다.

### 사용 기술

- **프론트엔드**: React (사용자 화면)
- **백엔드**: Spring Boot (서버 처리)
- **데이터베이스**: Oracle Database (데이터 저장)
- **포트**:
  - 프론트엔드: 5174번
  - 백엔드: 8090번

## 🔐 1. 로그인 기능

### 1.1 회원가입

사용자가 계정을 새로 만드는 기능입니다.

#### 입력 정보

- **아이디**: 다른 사람과 중복되지 않는 고유 아이디
- **비밀번호**: 본인 확인용 비밀번호
- **비밀번호 확인**: 입력 실수 방지를 위한 재입력
- **이름**: 실명 또는 닉네임
- **이메일**: 연락 가능한 이메일 주소
- **닉네임**: 게시물에 표시될 이름

#### 주요 기능

1. **아이디 중복 확인**
   - "중복 확인" 버튼을 클릭하면 이미 사용 중인 아이디인지 확인
   - 사용 가능한 아이디인 경우 "사용 가능한 아이디입니다" 메시지 표시
   - 중복된 아이디인 경우 "이미 사용 중인 아이디입니다" 메시지 표시

2. **비밀번호 일치 확인**
   - 비밀번호와 비밀번호 확인이 일치하지 않으면 경고 메시지 표시
   - 일치해야만 회원가입 진행 가능

3. **자동 날짜 기록**
   - 회원가입 시점을 자동으로 데이터베이스에 저장
   - 나중에 회원 가입일 확인 가능

#### 처리 과정

1. 사용자가 회원가입 폼에 정보 입력
2. "중복 확인" 버튼으로 아이디 사용 가능 여부 확인
3. 모든 정보 입력 후 "회원가입" 버튼 클릭
4. 서버에서 입력 정보 검증
5. 데이터베이스에 회원 정보 저장
6. 성공 시 로그인 페이지로 자동 이동

### 1.2 로그인

가입한 계정으로 서비스에 접속하는 기능입니다.

#### 입력 정보

- **아이디**: 회원가입 시 등록한 아이디
- **비밀번호**: 회원가입 시 설정한 비밀번호

#### 주요 기능

1. **계정 인증**
   - 아이디와 비밀번호가 데이터베이스에 저장된 정보와 일치하는지 확인
   - 일치하면 로그인 성공, 불일치하면 로그인 실패 메시지 표시

2. **로그인 상태 유지**
   - 로그인 성공 시 사용자 번호를 브라우저에 임시 저장
   - 이후 게시물 작성 등에서 본인 확인용으로 사용

3. **홈으로 이동**
   - 로그인 페이지 상단의 "Instagram" 로고 클릭 시 메인 페이지로 이동
   - 로그인하지 않고도 다른 사람의 게시물 구경 가능

#### 처리 과정

1. 사용자가 아이디와 비밀번호 입력
2. "로그인" 버튼 클릭
3. 서버에서 데이터베이스의 회원 정보와 비교
4. 일치하면 로그인 성공, 메인 페이지로 이동
5. 불일치하면 오류 메시지 표시

### 1.3 데이터 저장 구조

회원 정보는 다음과 같이 데이터베이스에 저장됩니다:

- **회원 번호**: 자동으로 부여되는 고유 번호
- **아이디**: 로그인용 ID (중복 불가)
- **비밀번호**: 계정 보안용
- **이름**: 사용자 실명
- **이메일**: 연락처 (중복 불가)
- **닉네임**: 게시물에 표시되는 이름
- **가입일**: 회원가입 날짜와 시간
- **수정일**: 정보가 마지막으로 변경된 날짜

## 📝 2. 게시물 작성 기능

### 2.1 게시물 작성 시작

메인 페이지 좌측 사이드바의 "만들기" 버튼을 클릭하면 게시물 작성 모달창이 열립니다.

### 2.2 이미지 업로드 (1단계)

게시물에 올릴 사진을 선택하는 단계입니다.

#### 주요 기능

1. **이미지 선택**
   - "컴퓨터에서 선택" 버튼 클릭
   - 컴퓨터에서 이미지 파일 선택 (jpg, png 등)

2. **미리보기**
   - 선택한 이미지가 화면에 즉시 표시
   - 사진 확인 후 "다음" 버튼으로 다음 단계 진행
   - 잘못 선택한 경우 "사진 바꾸기" 버튼으로 다시 선택

### 2.3 내용 작성 (2단계)

이미지와 함께 올릴 글과 위치를 입력하는 단계입니다.

#### 화면 구성

- **좌측**: 선택한 이미지 최종 미리보기 (흰색 배경)
- **우측**: 내용 입력 영역

#### 입력 정보

1. **문구 입력**
   - 최대 2,000자까지 입력 가능
   - 여러 줄 입력 가능 (줄바꿈 지원)
   - 긴 글도 자동으로 줄바꿈되어 표시
   - 흰색 배경에 검은색 글씨로 표시
   - 얇은 회색 테두리가 있는 입력창

2. **위치 추가** (선택사항)
   - 최대 100자까지 입력 가능
   - 예: "서울 강남구", "제주도 한라산" 등
   - 입력하지 않아도 게시물 작성 가능

#### 주요 기능

1. **뒤로 가기**
   - "뒤로" 버튼 클릭 시 1단계(이미지 선택)로 돌아가기
   - 다른 이미지로 변경 가능

2. **게시물 공유**
   - 모든 내용 입력 후 "공유하기" 버튼 클릭
   - 서버로 데이터 전송 및 저장
   - 성공 시 메인 페이지 자동 새로고침
   - 새로 작성한 게시물이 피드 맨 위에 표시

### 2.4 이미지 저장 방식

업로드된 이미지는 다음과 같이 처리됩니다:

1. **파일 이름 변경**
   - 원본 파일명 대신 고유한 이름 자동 생성
   - 예: "abc123-def456.jpg"
   - 파일 이름 중복 방지

2. **서버 저장**
   - 백엔드 서버의 `uploads` 폴더에 저장
   - 파일 경로를 데이터베이스에 기록

3. **화면 표시**
   - 피드에서 게시물을 볼 때 저장된 이미지 자동 로드
   - 주소 형식: `http://localhost:8090/uploads/파일명`

### 2.5 게시물 데이터 저장 구조

게시물 정보는 다음과 같이 데이터베이스에 저장됩니다:

- **게시물 번호**: 자동으로 부여되는 고유 번호
- **작성자 번호**: 게시물을 작성한 회원의 번호
- **이미지 경로**: 업로드된 이미지 파일의 저장 위치
- **내용**: 게시물의 글 내용
- **위치**: 게시물 작성 위치 (선택사항)
- **좋아요 수**: 0으로 자동 설정 (추후 기능 구현 예정)
- **작성일**: 게시물 작성 날짜와 시간 (자동 기록)
- **수정일**: 게시물이 마지막으로 변경된 날짜 (자동 기록)

### 2.6 피드에서 게시물 확인

메인 페이지에서 작성된 게시물들을 확인할 수 있습니다.

#### 표시 순서

- **최신순 정렬**: 가장 최근에 작성된 게시물이 맨 위에 표시
- 스크롤하여 이전 게시물들 확인 가능

#### 표시 내용

- **프로필 영역**: 작성자의 닉네임
- **이미지**: 업로드된 사진
- **좋아요/댓글 아이콘**: 향후 기능 구현 예정
- **내용**: 게시물의 글 내용 (긴 글은 자동 줄바꿈)

## 🎨 3. 사용자 인터페이스 특징

### 3.1 다크모드 대응

- 브라우저가 다크모드로 설정되어 있어도 입력창은 항상 밝은 배경 유지
- 흰색 배경 + 검은색 글씨로 고정
- 가독성 향상

### 3.2 반응형 디자인

- 입력창이 모달 밖으로 나가지 않도록 자동 조정
- 긴 글 작성 시 자동 줄바꿈
- 모바일 환경에서도 사용 가능 (반응형 레이아웃)

### 3.3 사용자 편의 기능

- 로고 클릭으로 메인 페이지 이동
- 시각적 피드백 (버튼 호버 효과)
- 명확한 오류 메시지 표시
- 입력 제한 (글자 수, 파일 형식 등)

## 💾 5. 데이터 흐름

### 회원가입 흐름

```
사용자 입력 → 프론트엔드 검증 → 백엔드 전송 → 데이터베이스 저장 → 성공 응답 → 로그인 페이지 이동
```

### 로그인 흐름

```
아이디/비밀번호 입력 → 백엔드 인증 → 데이터베이스 조회 → 일치 확인 → 로그인 성공 → 메인 페이지 이동
```

### 게시물 작성 흐름

```
이미지 선택 → 미리보기 → 내용 입력 → 서버 전송 → 이미지 저장 → 데이터베이스 저장 → 피드 새로고침
```

### 게시물 조회 흐름

```
메인 페이지 접속 → 백엔드 요청 → 데이터베이스 조회 → 최신순 정렬 → 프론트엔드 표시 → 이미지 로드
```

## 🔧 6. 기술적 구현 방식

### 6.1 프론트엔드 (React)

- **컴포넌트 구조**: Login, Signup, Feed, Sidebar, CreatePostModal 등으로 분리
- **상태 관리**: React useState와 useEffect 훅 사용
- **API 통신**: fetch API로 백엔드와 HTTP 통신
- **라우팅**: React Router로 페이지 이동 관리

### 6.2 백엔드 (Spring Boot)

- **REST API**: GET, POST 방식의 HTTP API 제공
- **파일 업로드**: MultipartFile로 이미지 처리
- **데이터베이스 연동**: JPA/Hibernate로 Oracle DB 연결
- **자동 ID 생성**: 데이터베이스 시퀀스 사용

### 6.3 데이터베이스 (Oracle)

- **USERS 테이블**: 회원 정보 저장
- **POSTS 테이블**: 게시물 정보 저장
- **시퀀스**: 자동 증가 번호 생성 (SEQ_USERS_NO, SEQ_POSTS_NO)
- **관계**: POSTS → USERS (작성자 연결)
